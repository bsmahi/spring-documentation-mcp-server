Tips
Username

Choose a unique username. It must be alphanumeric and can include dots (.), underscores (_), and hyphens (-).

Password Security

Passwords must be at least 8 characters long. Use a combination of letters, numbers, and special characters for better security.

User Roles
  • ADMIN: Can manage all aspects of the system including users, projects, versions, and documentation.
  • VIEWER: View-only access to the system. Can browse documentation and projects but cannot make any modifications.
Enabled Status

Unchecking "Enabled" will prevent the user from logging in without deleting their account. This is useful for temporary suspensions.

Security Note: Be careful when assigning the ADMIN role. Admins have full control over the system.
