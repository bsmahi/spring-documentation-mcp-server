Tips
Version String

Use the official version string from spring.io (e.g., "3.5.7", "3.5.0-RC1", "3.6.0-SNAPSHOT")

Version Numbers

Break down the version into major, minor, and patch components. For example, "3.5.7" = Major: 3, Minor: 5, Patch: 7

Version Types
  • STABLE: Production-ready release
  • RC: Release candidate for testing
  • SNAPSHOT: Development version
  • MILESTONE: Feature-complete but not final
Flags

Mark a version as "Latest" if it's the most recent stable release. Mark as "Default" if it should be the primary documentation version.

Note: Only one version per project should be marked as "Latest" and "Default" at a time.
Examples
Spring Boot 3.5.7 (STABLE)
  • Version: 3.5.7
  • Major: 3, Minor: 5, Patch: 7
  • Type: STABLE
Spring Framework 6.2.0-RC1
  • Version: 6.2.0-RC1
  • Major: 6, Minor: 2, Patch: 0
  • Type: RC
Spring Data 2024.1.0-SNAPSHOT
  • Version: 2024.1.0-SNAPSHOT
  • Major: 2024, Minor: 1, Patch: 0
  • Type: SNAPSHOT
